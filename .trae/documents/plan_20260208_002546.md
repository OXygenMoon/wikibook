**Problem:**
The `if __name__ == "__main__":` block (which starts the Flask server) is located at lines 3353-3358, but there are new route definitions (`service_wiki_pages` and `view_page`) **after** this block (lines 3360-3451).
When the application runs, it blocks at `app.run()`, meaning the code following it is never executed. This causes the `view_page` route (and others) to be unregistered, leading to `BuildError` (500 Internal Server Error) when other pages try to generate URLs for them, or 404/502 errors depending on how the server is accessed.

**Plan:**
1.  **Move the Main Block**: I will move the following code block from the middle of `app.py` to the very end of the file:
    ```python
    if __name__ == "__main__":
        with app.app_context():
            db.create_all()
            upgrade_db() # Run upgrade check
            
        app.run(host="0.0.0.0", port=int(os.environ.get("PORT", "5009")), debug=True)
    ```
2.  **Verify**: This ensures all routes are registered before the server starts.

**需求分析**

用户希望调整贴纸库的交互逻辑和层级关系：

1. **半折叠状态 (Partial Collapse)**：

   * **当前行为**：拖动贴纸时，抽屉完全收起（`translateY(100%)`）。

   * **期望行为**：拖动贴纸时，抽屉**下滑但不要完全消失**，留出一部分（标题栏或把手）可见。

   * **操作恢复**：

     * 当贴纸拖拽结束（`mouseup`/`touchend`）时，抽屉应**自动上滑恢复**。

     * 增加一个“上滑下滑按钮”（实际上现有的标题栏或新增把手可以充当此功能），或者利用留出的部分点击恢复。

     * 注意：用户之前提过“布局好后可以点击某一个展开按钮”，现在又说“移动到位后窗体会上滑”。这两者有细微冲突，需要权衡。结合“增加一个窗体上滑下滑按钮”，合理的逻辑是：

       * 拖拽开始 -> 抽屉半折叠。

       * 拖拽结束 -> 抽屉**保持半折叠**（方便继续调整）还是**自动恢复**？用户说“当把贴纸移动到位以后，窗体会上滑”，这暗示是自动恢复。但如果用户要连续拖动多个贴纸，自动恢复会很烦。

       * **修正理解**：用户原话是“当把贴纸移动到位以后，窗体会上滑”。这意味着拖拽释放后自动恢复。

       * **按钮需求**：用户提到“增加一个窗体上滑下滑的按钮”，可能是指手动控制的开关，或者是指半折叠状态下留出的那个“把手”本身就是一个按钮。

2. **层级调整 (Z-Index)**：

   * **当前行为**：贴纸可能遮挡了顶部的搜索栏、精选 Wiki 等元素。

   * **期望行为**：贴纸的高度（层级）要低于搜索栏和精选 Wiki 窗体。

   * **实现**：调整 `#sticker-canvas` 的 `z-index`。目前是 900（较高）。Navbar 通常是 1000 或 40/50（Tailwind）。需要检查 `base.html` 中 Navbar 的层级，确保 `#sticker-canvas` 低于它。

**解决方案设计**

1. **样式修改 (`sticker.css`)**：

   * 新增 `.sticker-drawer.partial` 类：`transform: translateY(calc(100% - 60px))`，保留 60px 高度。

   * 调整 `#sticker-canvas` 的 `z-index`：设为 `20`（假设 Navbar 是 40 或 50，Content 是 10）。这样贴纸在背景之上，但在悬浮 UI 之下。

2. **逻辑修改 (`sticker.js`)**：

   * **`collapseDrawer()`**: 改为添加 `.partial` 类，而不是完全移除 `.open`。同时确保 FAB 按钮隐藏（因为此时抽屉头部还在）。

   * **`expandDrawer()`**: 移除 `.partial`，添加 `.open`。

   * **拖拽事件**：

     * `mousedown`/`touchstart`: 调用 `collapseDrawer()`。

     * `mouseup`/`touchend`: 调用 `expandDrawer()`。**注意**：用户明确说“移动到位以后，窗体会上滑”。这意味着松手即恢复。

   * **手动按钮**：在抽屉头部添加一个显式的“折叠/展开”图标按钮（Chevron Down/Up），或者让整个头部点击可切换。

**实施步骤**

1. **修改 CSS**：定义 `.partial` 状态和调整 `z-index`。
2. **修改 JS**：

   * 更新 `collapseDrawer` / `expandDrawer` 逻辑。

   * 在拖拽结束监听器中添加 `expandDrawer()` 调用。

   * 给 Drawer Header 添加点击事件：如果处于 `partial` 状态，点击则 `expand`。

* `sticker.css`: `#sticker-canvas`, `.sticker-drawer`

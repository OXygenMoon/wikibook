# 关注与学友功能实现计划

## 1. 数据库模型变更
需要新建一个多对多关联表来存储用户之间的关注关系。

### 新增 `UserFollow` 模型（或自关联表）
在 `app.py` 中新增 `followers` 关联表：
- **follower_id**: 关注者 ID (Integer, ForeignKey('user.id'))
- **followed_id**: 被关注者 ID (Integer, ForeignKey('user.id'))
- **created_at**: 关注时间 (DateTime)

### User 模型更新
在 `User` 类中添加 `followed` 和 `followers` 关系属性：
- `following`: 我关注的人
- `followers`: 关注我的人
- 辅助方法 `is_following(user)`: 检查是否关注某人
- 辅助方法 `is_mutual_following(user)`: 检查是否互相关注（即“学友”关系）

## 2. 后端逻辑实现

### 新增路由
- `POST /user/<int:user_id>/follow`: 关注用户
- `POST /user/<int:user_id>/unfollow`: 取消关注用户
- `GET /user/<int:user_id>/following`: 查看某人的关注列表
- `GET /user/<int:user_id>/followers`: 查看某人的粉丝列表（仅显示计数，具体名单不可见，除非是自己查看）

### 业务逻辑
- **关注逻辑**：
  - A 关注 B -> B 的粉丝数+1，A 的关注数+1。
  - 检查是否 B 也关注了 A -> 如果是，标记为“学友”关系。
- **学友逻辑**：
  - 互相关注即为学友。
  - 学友之间可以查看对方在线状态（基于现有的 `UserActiveStatus`）。
- **消息推送逻辑（基于轮询的模拟推送）**：
  - 由于系统无 WebSocket，我们将利用现有的 `heartbeat` 接口或新增 `api/notifications` 轮询接口。
  - 新增 `Notification` 模型存储消息：
    - `recipient_id`: 接收者
    - `content`: 消息内容
    - `is_read`: 是否已读
    - `type`: 类型（上线提醒、成就提醒等）
  - **触发点**：
    - 用户登录时 (`login_user`) -> 给所有在线学友创建 Notification。
    - 获得成就时 (`check_and_award_badges`) -> 给所有在线学友创建 Notification。
    - 用户下线（Session过期或主动退出） -> 较难实时捕获，暂略或基于心跳超时判定。

## 3. 前端页面修改

### [profile.html](file:///root/projects/wikibook/templates/user/profile.html)
- **新增统计数据**：在姓名下方添加 "x 学友 | x 关注 | x 被关注"。
  - 学友数：互相关注的人数。
  - 关注数：我关注的人数。
  - 被关注数：关注我的人数。
- **关注按钮**：
  - 如果查看他人主页，显示“关注/取消关注”按钮。
  - 如果互相关注，显示“学友”标识。

### [online_users.html](file:///root/projects/wikibook/templates/online_users.html)
- 列表项添加链接跳转到个人主页。
- 列表项添加“关注/取消关注”按钮（AJAX 操作）。

### 新增关注列表页面 `user/following.html`
- 展示我关注的用户列表。
- 显示对方状态（是否互相关注/学友）。
- 点击头像/名字跳转到对方主页。

### 全局通知展示 (`base.html`)
- 在页面顶部或右下角添加通知提示区域。
- 前端定时（如每30秒，配合心跳）请求未读消息并弹窗显示。

## 4. 实施步骤
1.  **数据库迁移**：定义 `followers` 表，更新 `User` 模型，执行 `db.create_all()`。
2.  **后端接口**：实现关注/取消关注 API，实现通知查询 API。
3.  **模板修改**：修改 `profile.html` 和 `online_users.html`，增加 UI 元素。
4.  **通知机制**：实现简单的 Notification 存储与轮询获取逻辑。
5.  **验证**：测试关注流程、学友判定及消息通知。

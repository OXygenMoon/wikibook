# HTML Refactoring Plan: Profile Page (GitHub Redesign Style)

I will completely refactor `templates/user/profile.html` to match the provided GitHub Redesign concept art, while strictly adhering to the "White Theme" requirement.

## 1. Layout Structure (Bento Grid)
The new layout will abandon the current "Header + Rows" structure and adopt a **Two-Column Layout**:
-   **Left Sidebar (Fixed/Sticky)**: User Info, Navigation, Social Stats.
-   **Right Main Content (Bento Grid)**: A responsive grid system holding various functional cards.

### Grid Composition (Right Column)
Using CSS Grid (`grid-cols-12`):
1.  **Top Row**:
    -   **Hello/Intro Card (8 cols)**: Replaces "Readme". Shows a greeting, bio, or pinned note.
    -   **Avatar/Feature Image (4 cols)**: Large visual element (current profile header gloss effect can be adapted here).
2.  **Middle Row**:
    -   **Achievements (4 cols)**: The "Badge Wall" preview.
    -   **Contribution Graph (8 cols)**: The Heatmap.
3.  **Bottom Row**:
    -   **Popular Repositories (6 cols)**: "Recent Notes" or "Wiki Contributions".
    -   **Stats/Habits (6 cols)**: The Radar Chart or other stats.

## 2. Visual Style (White Theme Adaptation)
Since the reference is dark mode, I will translate it to a clean white theme:
-   **Backgrounds**: `bg-white` (or very light gray `bg-base-200` for the page background to make white cards pop).
-   **Cards**: `bg-white`, `rounded-3xl` (matching the reference's heavy rounding), `shadow-sm`, `border border-base-200`.
-   **Typography**: Heavy use of bold headings, clean sans-serif.
-   **Spacing**: Generous padding (`p-6` or `p-8`).

## 3. Component Details

### A. Left Sidebar
-   **User Identity**: Avatar (Rounded square or circle), Name (Large), Username (Gray).
-   **Vertical Navigation**: "Overview", "Notes", "Badges", "Network".
    -   *Action*: I will convert the current top tabs or links into this vertical list.
-   **Metadata**: Location, Join Date, etc., at the bottom.
-   **Social**: Followers/Following count with a big "Follow" button at the bottom.

### B. Bento Cards
1.  **Intro Card**: "Howdy!" style text using `user.bio` (if exists) or a default greeting.
2.  **Heatmap Card**: Port the existing ECharts heatmap into a card that spans 2/3 of the width.
3.  **Badge Card**: A grid of circular badges (Achievement style).
4.  **Notes Card**: A list of recent notes with "Public Archive" style tags.

## 4. Implementation Steps
1.  **Backup**: Ensure current logic is preserved (I will comment out old code or replace carefully).
2.  **Scaffold**: Create the new `div.flex` container with `aside` and `main`.
3.  **Migrate Components**:
    -   Move `heatmap` logic to the new Heatmap Card.
    -   Move `user_badges` logic to the Achievements Card.
    -   Move `recent_notes` logic to the Repositories Card.
    -   Move `stats` (radar chart) to a new Stats Card.
4.  **Styling**: Apply `rounded-[2rem]`, `shadow-sm`, and `hover:shadow-md` transitions to match the "tactile" feel of the reference.
5.  **Responsiveness**: Stack everything on mobile, side-by-side on desktop.

*Note: I will not modify `base.html` significantly to avoid breaking other pages, but I will ensure `profile.html` overrides any constraints if necessary using internal CSS or utility classes.*

## 目标
新增一个用户个人界面 (`/user/profile`)，整合用户信息、统计数据和成就列表（原“我的徽章”页面），并在左侧导航栏的个人信息区域提供入口。

## 变更范围
1.  **新增页面**: `templates/user/profile.html`
2.  **后端路由**: `app.py` 中添加 `/user/profile` 路由。
3.  **导航栏更新**: `templates/base.html` 中修改左下角个人信息区域的点击跳转链接。
4.  **样式调整**: 复用现有 CSS 样式，确保新页面风格统一。

## 详细计划

### 1. 后端路由 (`app.py`)
- 新增 `/user/profile` 路由。
- **数据获取**:
    - **用户信息**: `current_user`。
    - **统计数据**:
        - 创建的 Wiki 数量
        - 创建的笔记数量
        - 获得的徽章数量
        - 连续打卡天数 (复用 `streak_days` 逻辑)
    - **成就列表**:
        - 获取已获得徽章 (`UserBadge`)。
        - 获取未获得徽章 (`Badge`)。
- **渲染模板**: `render_template("user/profile.html", ...)`。

### 2. 前端模板 (`templates/user/profile.html`)
- **布局结构**:
    - **头部 (Header)**: 用户头像、用户名、身份标签（管理员/用户）、个人简介（预留，暂显示“这个人很懒...”）。
    - **统计卡片 (Stats)**: 4列网格，显示“Wiki数”、“笔记数”、“徽章数”、“打卡天数”。
    - **主要内容区 (Main Content)**:
        - **选项卡 (Tabs)**:
            - **“我的成就”**: 整合原 `my_badges.html` 内容，显示已获得和未获得的徽章，支持佩戴功能。
            - **“近期动态”**: (可选) 显示最近创建的 Wiki 或笔记。
- **样式**: 使用 Tailwind CSS + DaisyUI，保持与全站一致的清新风格。

### 3. 导航栏修改 (`templates/base.html`)
- 定位到侧边栏底部 (`.sidebar-content` 下方)。
- 找到包含用户头像和用户名的 `div`。
- 修改 `onclick` 事件：从 `window.location.href='{{ url_for('my_badges') }}'` 改为 `window.location.href='{{ url_for('user_profile') }}'`。
- (可选) 将原来的“我的成就”相关逻辑完全迁移到新页面后，可以考虑废弃单独的 `/user/badges` 路由，或者将其重定向到个人主页。

### 4. 迁移与整合
- 将 `templates/user/my_badges.html` 中的徽章展示逻辑（包括佩戴/卸下 JS 代码）复制到 `profile.html` 中。
- 确保徽章图标、提示框（Tooltip）等功能正常工作。

## 预期效果
用户点击左下角头像区域时，不再跳转到单纯的徽章列表，而是进入一个功能更丰富的个人主页，既能看到自己的成就，也能概览自己在平台上的贡献和活跃度。

**设计方案：统一徽章容器设计**

为了解决不同图标（Emoji vs 图片、长宽比不一的图片）显示效果不一致的问题，建议采用\*\*“统一容器 + 居中缩放”\*\*的方案。这类似于 App 图标的设计思路，强制将所有徽章放入一个固定尺寸的容器中，无论原始内容如何，都能获得一致的视觉重量和对齐效果。

**具体方案**

1. **统一容器样式**

   * 为所有徽章（无论是侧边栏还是在线列表）增加一个圆角正方形或圆形背景容器。

   * 使用 `flex` 布局保证内容绝对居中。

   * 统一背景色（如浅色半透明背景），增强图标的辨识度，同时也能作为视觉边界，让不同形状的图片看起来更整齐。

2. **图片优化**

   * **尺寸限制**：强制图片的最大宽/高为容器的 80%\~90%，保留一定的内边距（Padding），防止图片撑满边缘显得拥挤。

   * **对象适应**：继续使用 `object-contain` 保持原比例，确保图片不被拉伸。

3. **Emoji/文字优化**

   * **字号统一**：设定与容器相匹配的固定 `font-size` 和 `line-height`。

   * **居中对齐**：通过 Flexbox 完美居中，消除基线对齐带来的微小偏差。

**实施步骤**

1. **创建通用 CSS 类**

   * 在 `base.html` 或新建 CSS 文件中定义 `.badge-icon-container` 类。

   * 样式示例：

     ```css
     .badge-icon-container {
         display: inline-flex;
         align-items: center;
         justify-content: center;
         background-color: rgba(0, 0, 0, 0.05); /* 极淡的背景 */
         border-radius: 6px; /* 圆角 */
         overflow: hidden;
         vertical-align: middle;
         transition: transform 0.2s;
     }
     .badge-icon-container:hover {
         transform: scale(1.1); /* 悬停微放大效果 */
         background-color: rgba(0, 0, 0, 0.1);
     }
     .badge-icon-container img {
         width: 85%;
         height: 85%;
         object-fit: contain;
     }
     ```

2. **修改** **`templates/base.html`**

   * 将原本直接裸露的 `<img>` 或 `<span>` 包裹在新的 `.badge-icon-container` 中。

   * 设定尺寸为 `w-6 h-6` (24px)，略微增大以容纳内边距。

3. **修改** **`templates/online_users.html`**

   * 同样应用容器结构。

   * 设定尺寸为 `w-5 h-5` (20px)，保持紧凑。

**预期效果**

* 无论用户上传的是长方形、正方形还是透明背景的图片，都会被限制在一个整齐的圆角方块内。

* Emoji 和图片在视觉上高度对齐，不会出现忽大忽小或基线跳动的情况。

* 增加的淡色背景能让透明 PNG 图片在不同主题色下都能看清。

**功能设计**

目前的 DaisyUI `tooltip` 组件仅支持纯文本 (`data-tip`)，无法满足“显示清晰大图、介绍和获得时间”这种富文本内容的需求。

因此，我将弃用原有的 CSS Tooltip，转而使用一个**自定义的 Popover（悬浮卡片）组件**。

**交互逻辑**

1. **触发**：鼠标悬停在徽章小图标上。
2. **显示**：在图标旁弹出一个层级最高的悬浮卡片（Popover）。
3. **内容**：

   * **头部**：放大版的徽章图标（清晰可见）+ 徽章名称。

   * **中部**：徽章的描述文本。

   * **底部**：获得时间（格式化为 `YYYY-MM-DD`）。
4. **消失**：鼠标移出图标。

**技术实现**

1. **数据获取**：

   * `base.html` (侧边栏)：需要获取当前用户佩戴徽章的获得时间。目前 `current_user.selected_badge` 只是 `Badge` 对象，不包含 `earned_at`。我需要在 `app.py` 的 `User` 模型中增加一个 helper 方法或属性，或者在模板中通过查询获取（稍显复杂）。

   * **更优解**：修改 `User.selected_badge` 的关联方式，或者在 `User` 模型上添加一个 `selected_user_badge` 属性，直接指向 `UserBadge` 记录，这样就能同时拿到 `Badge` 信息和 `earned_at`。

   * `online_users.html`：同理，需要获取 `UserBadge` 对象而不仅仅是 `Badge`。

2. **前端组件**：

   * 创建一个隐藏的模板结构（Popover），使用绝对定位。

   * 使用少量的 JavaScript (或者纯 CSS `group-hover`) 来控制显示/隐藏。为了保证定位准确（防止被父容器 `overflow: hidden` 遮挡），建议使用 JavaScript 动态计算位置，或者使用像 Tippy.js 这样的库（如果项目已有），否则简单的 CSS `absolute` 定位即可（需注意父级 `z-index`）。

   * 考虑到项目已有 Tailwind，我将使用 Tailwind 的 `group` 和 `group-hover` 实现纯 CSS 的富文本 Tooltip，这是最轻量且性能最好的方案。

**后端模型调整 (`app.py`)**

* 目前 `User.selected_badge_id` 只是存 ID。

* `User.selected_badge` 是 `relationship("Badge")`。

* 我需要在 `User` 模型中添加一个方法 `get_selected_user_badge()`，它返回对应的 `UserBadge` 实例，以便获取 `earned_at`。

**实施步骤**

1. **后端**：在 `app.py` 的 `User` 模型中添加 `get_selected_user_badge()` 方法。
2. **前端 (`base.html`** **&** **`online_users.html`)**：

   * 修改徽章的 HTML 结构，包裹在一个 `group relative` 容器中。

   * 在容器内添加一个 `absolute` 定位的详细信息卡片 (`invisible group-hover:visible`)。

   * 卡片内布局：大图、名称、描述、时间。

## 问题原因
贴纸向右移动时自动缩小，是由于 Tailwind CSS 的默认样式（Preflight）给所有 `img` 设置了 `max-width: 100%`。当贴纸（绝对定位元素）靠近页面右边缘时，其父容器的可用宽度变小，导致内部图片受限于父容器宽度而被迫缩小。

## 修复方案
我们需要修改 [sticker.css](file:///root/projects/wikibook/static/css/sticker.css) 来覆盖这一默认行为，确保贴纸尺寸固定，不受页面布局限制。

### 1. 修改 `.sticker-item`
- 添加 `width: max-content;`：确保容器宽度由内容决定，而不是由其位置（剩余空间）决定。

### 2. 修改 `.sticker-item img`
- 添加 `max-width: none;`：强制覆盖全局的 `max-width: 100%` 限制，允许图片保持原尺寸，即使超出父容器或视口边缘。

## 代码变更预览
**文件**: `static/css/sticker.css`

```css
.sticker-item {
    /* ...原有样式... */
    width: max-content; /* 新增：防止容器被挤压 */
}

.sticker-item img, .sticker-item .sticker-text {
    /* ...原有样式... */
    max-width: none; /* 新增：解除最大宽度限制 */
}
```

修复后，无论将贴纸拖动到页面的任何位置（包括边缘），它都将保持原始尺寸，不会发生形变或缩小。

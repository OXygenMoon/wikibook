## 目标
实现查看其他用户个人主页的功能，并确保能看到他们布置的贴纸。

## 变更范围
1.  **后端路由 (`app.py`)**:
    *   新增 `/user/<int:user_id>` 路由，用于访问指定用户的公开主页。
    *   修改 `/api/stickers/<page_type>` 接口，支持通过 `user_id` 参数获取指定用户的贴纸数据。
2.  **前端模板 (`templates/user/profile.html`)**:
    *   修改模板以支持渲染非当前用户的数据（使用 `user` 变量替代硬编码的 `current_user`）。
    *   当查看他人主页时，隐藏编辑控件（如佩戴徽章、修改贴纸等）。
3.  **前端脚本 (`static/js/sticker.js`)**:
    *   更新 `StickerManager`，使其支持在初始化时传入 `userId` 参数，以便从 API 获取正确用户的贴纸。

## 详细计划

### 1. 后端改造 (`app.py`)
*   **修改 Sticker API**:
    *   在 `api_stickers` 中，添加 `target_user_id` 逻辑。如果是 GET 请求且 URL 包含 `user_id` 参数，则查询该用户的 `UserSticker`。
*   **新增公开主页路由**:
    *   添加 `@app.route("/user/<int:user_id>")`。
    *   逻辑与 `/user/profile` 类似，但获取的是 `user_id` 对应的 `User` 对象。
    *   **权限控制**: 如果访问的是自己的 ID，重定向到 `/user/profile`（或直接复用逻辑）。
    *   传递给模板的变量名统一为 `user`，在模板中区分 `user` (目标用户) 和 `current_user` (当前登录用户)。

### 2. 模板通用化 (`templates/user/profile.html`)
*   将模板中所有的 `current_user` 替换为 `user`（除了判断是否为“我自己”的逻辑）。
*   **只读模式**:
    *   如果是查看他人主页 (`user.id != current_user.id`)：
        *   隐藏“拖拽贴纸到这里”提示。
        *   隐藏徽章佩戴功能的点击事件。
        *   初始化 `StickerManager` 时传入 `user.id`。

### 3. 前端脚本更新 (`static/js/sticker.js`)
*   修改 `StickerManager` 的构造函数，接受 `targetUserId` 参数。
*   在 `loadStickers` 方法中，发起 fetch 请求时附带 `?user_id=${this.targetUserId}`。

## 预期效果
*   用户访问 `/user/123` 可以看到 ID 为 123 的用户的个人主页。
*   主页顶部会显示该用户在首页布置的贴纸。
*   用户无法修改他人的贴纸或佩戴他人的徽章。

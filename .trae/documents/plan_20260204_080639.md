## 目标
实现贴纸拖拽时的即时视觉反馈：
1. **绿色边框**：当贴纸超过一半面积进入“贴纸放置提示区域”时。
2. **红色边框**：当贴纸不足一半面积在区域内（但仍在拖拽中）时。
3. **允许放置**：无论是否在区域内，松开鼠标均允许放置（不强制回弹）。

## 变更范围
- 前端脚本：[static/js/sticker.js](file:///root/projects/wikibook/static/js/sticker.js)

## 实现方案
- 在 `StickerManager` 类中增加碰撞检测逻辑：
  1. 获取提示区域 (`#sticker-drop-zone`) 的位置尺寸 (`getBoundingClientRect`)。
  2. 获取当前拖拽贴纸的位置尺寸。
  3. 计算两者的**相交面积**。
  4. 计算贴纸的总面积。
  5. 判断：`相交面积 / 贴纸面积 > 0.5`。
- 修改拖拽事件监听器（`mousemove` 和 `touchmove`）：
  - 实时执行上述检测。
  - 根据结果动态添加/移除 CSS 类（如 `valid-drop` 为绿框，`invalid-drop` 为红框）。
- 定义 CSS 样式（可直接通过 JS 操作 style 边框颜色，或添加 Tailwind 类）：
  - 绿色状态：`border: 2px solid #22c55e` (Tailwind `green-500`)
  - 红色状态：`border: 2px solid #ef4444` (Tailwind `red-500`)
- 拖拽结束（`mouseup` / `touchend`）时：
  - 移除所有临时的状态类/样式，恢复原状。

## 关键代码逻辑
```javascript
// 在拖拽移动循环中
const stickerRect = el.getBoundingClientRect();
const zoneRect = dropZone.getBoundingClientRect();

// 计算相交矩形
const intersectionW = Math.max(0, Math.min(stickerRect.right, zoneRect.right) - Math.max(stickerRect.left, zoneRect.left));
const intersectionH = Math.max(0, Math.min(stickerRect.bottom, zoneRect.bottom) - Math.max(stickerRect.top, zoneRect.top));
const intersectionArea = intersectionW * intersectionH;

// 贴纸面积
const stickerArea = stickerRect.width * stickerRect.height;

// 判断
if (intersectionArea > stickerArea * 0.5) {
    el.style.border = '2px solid #22c55e'; // Green
} else {
    el.style.border = '2px solid #ef4444'; // Red
}
```

## 补充说明
- 提示区域 `#sticker-drop-zone` 默认可能包含子元素，检测时应针对其可视的外框容器。
- 考虑到旋转 (`rotation`) 可能会让矩形计算变复杂，为简化性能开销，初期方案暂按未旋转的边界盒（bounding box）计算，这在大多数 UX 场景下已足够准确。
- 现有的 `makeInteractable` 方法中的拖拽逻辑将被增强。

# Social Sticker Board Implementation Plan

I will implement the "Sticker Socialization" feature in two phases to ensure a smooth transition from a personal tool to a social experience.

## Phase 1: Backend & Data Model
1.  **Enhance Sticker API (`app.py`)**:
    -   Modify `GET /api/stickers/<page_type>` to support social access:
        -   Allow users to fetch stickers of users they follow.
        -   Allow fetching if the board is public (future-proofing).
    -   Create `StickerBoardSnapshot` model to store "published" versions of the board.
    -   Add `POST /api/stickers/<page_type>/publish` to save the current arrangement as a snapshot.
    -   Add `GET /api/stickers/snapshots` to list user snapshots.

## Phase 2: Frontend Interaction
1.  **Update Sticker Engine (`sticker.js`)**:
    -   Add `readOnly` mode to `StickerManager`.
    -   In `readOnly` mode:
        -   Disable dragging, rotating, and scaling.
        -   Hide delete buttons and control handles.
        -   Hide the sticker drawer and floating action button (FAB).
2.  **Board View (`index.html`)**:
    -   Detect `user_id` in the URL query parameters.
    -   **Visitor Mode**:
        -   If `user_id` is present and not the current user, initialize `StickerManager` in `readOnly` mode.
        -   Display a banner "Visiting [User]'s Board".
    -   **Owner Mode**:
        -   Add a "Publish" button to the toolbar (next to "Save") to let users share their current setup.
3.  **Entry Points (`following.html`)**:
    -   Add a "View Sticker Board" button to the user cards in the "Following" list, linking to `/?user_id=TARGET_ID`.

This plan ensures that users can immediately start visiting the boards of people they follow, and board owners can "publish" specific versions they are proud of.

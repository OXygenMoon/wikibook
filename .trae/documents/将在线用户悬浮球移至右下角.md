**功能设计**

1. **数据模型 (`UserSticker`)**：

   * 在 `app.py` 中新增 `UserSticker` 模型，用于持久化存储用户在首页放置的贴纸信息。

   * 字段包括：`user_id`（关联用户）、`badge_id`（关联徽章图标）、`page_type`（区分 'wiki' 或 'book' 首页）、`x`（横坐标）、`y`（纵坐标）、`rotation`（旋转角度）、`scale`（缩放比例）、`z_index`（层级）。

   * 新增 API 接口：

     * `GET /api/stickers/<page_type>`: 获取当前用户在该页面的所有贴纸。

     * `POST /api/stickers/<page_type>`: 保存当前页面的贴纸状态（全量保存）。

2. **前端交互界面**：

   * **入口按钮**：在页面右下角悬浮一个“贴纸库”按钮（FAB）。

   * **贴纸库面板**：点击按钮从底部滑出面板，展示用户已获得的徽章列表（`current_user.earned_badges`）。

   * **画布层**：在页面内容之上（或作为背景层但可交互）添加一个透明容器，用于显示和操作贴纸。

3. **交互逻辑 (`sticker.js`)**：

   * **放置**：点击贴纸库中的徽章，若画布上没有则在屏幕中央生成；若已有则移除（Toggle 逻辑）。

   * **操作**：

     * **拖拽**：按住贴纸主体可自由拖动位置。

     * **选中态**：点击贴纸进入“选中”状态，显示边框和操作手柄。

     * **缩放/旋转**：通过拖动角落的手柄同时进行缩放和旋转（模拟主流设计工具或 Sticker 应用的体验）。

     * **移除**：双击贴纸，或在选中状态下点击移除按钮/按 Delete 键移除。

   * **保存**：点击面板上的“保存”按钮，将当前画布上所有贴纸的状态发送至后端保存，并收起面板。

4. **页面集成**：

   * 修改 `templates/index.html` (Wiki首页) 和 `templates/book/index.html` (笔记首页)。

   * 引入新的 CSS 样式和 JS 脚本。

   * 利用 Jinja2 模板传入当前用户的已获徽章数据。

**实施步骤**

1. **后端开发**：定义 `UserSticker` 模型，创建数据库表（自动迁移），实现 API 接口。
2. **前端组件**：编写 `sticker.css` (定义面板动画、贴纸样式、手柄样式) 和 `sticker.js` (核心交互逻辑)。
3. **模板集成**：在两个首页模板中插入 HTML 结构（按钮、面板容器、画布容器）并引入静态资源。
4. **调试与验证**：测试贴纸的增删改查、位置同步及不同页面的隔离性。

**技术细节**

* **手势/操作**：使用原生 JS 实现拖拽和基于中心点的旋转/缩放计算。

* **坐标系**：使用百分比坐标存储位置，以适应不同屏幕尺寸（响应式适配）。

* **图标来源**：复用现有的 `badge_icon_url` 过滤器获取高清图片。

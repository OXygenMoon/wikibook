# 修复徽章图标渲染问题

## 问题原因

之前的实现仅在 `manage_badges.html` 和 `badge_members.html` 中处理了图片路径的渲染。其他页面（侧边栏、在线用户列表、贴纸系统、成就墙）仍然直接输出 `badge.icon` 字段，当该字段存储的是文件路径（如 `uploads/badges/xxx.png`）时，浏览器会直接显示文本路径，而不是 `<img>` 标签。

## 修复计划

### 1. 全局模板修复

需要检查并更新以下模板文件，添加判断逻辑：如果 `icon` 以 `uploads/` 开头，渲染为 `<img>`，否则渲染为文本（Emoji）。

* **`templates/base.html`** (侧边栏用户信息区域)

  * 定位：`current_user.selected_badge.icon` 的输出位置。

  * 修改：添加 `{% if ... startswith('uploads/') %}` 判断。

* **`templates/online_users.html`** (在线用户列表)

  * 定位：用户名前面的徽章显示区域。

  * 修改：同上。

* **`templates/user/my_badges.html`** (我的成就墙)

  * 定位：已获得成就和待解锁成就的图标显示区域。

  * 修改：同上。

### 2. JavaScript 逻辑修复 (贴纸系统)

贴纸系统是通过 JS 动态渲染的，因此需要修改 `base.html` 中的 JS 代码。

* **`templates/base.html`** (贴纸脚本部分)

  * **`createStickerElement`** **函数**：生成放置在屏幕上的贴纸 DOM。

    * 修改：判断 `sticker.icon` 是否包含 `uploads/`。如果是，创建 `img` 元素；否则保持文本内容。

  * **`renderLibrary`** **函数**：生成贴纸库中的预览 DOM。

    * 修改：同上，根据图标类型动态生成 `img` 或 `div` 内容。

## 3. 实现细节

对于模板中的判断逻辑，统一使用：

```html
{% if badge.icon.startswith('uploads/') %}
    <img src="{{ url_for('static', filename=badge.icon) }}" class="..." alt="...">
{% else %}
    {{ badge.icon }}
{% endif %}
```

对于 JS 中的判断逻辑：

```javascript
if (icon.startsWith('uploads/')) {
    // Render <img> tag with /static/ prefix
} else {
    // Render text
}
```
